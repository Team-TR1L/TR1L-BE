# Operations Index
> TR1L 운영 문서는 “잘 됐을 때”가 아니라 "깨졌을 때 어떻게 복구하는지(Recovery)"를 보여준다.

# 운영 원칙
- 장애 대응은 **증상 → 원인 → 조치**로 기록
- 재실행(rerun)은 안전하게(중복/부분 처리 방지)
- 지표가 없으면 운영이 아니다(Observability)
