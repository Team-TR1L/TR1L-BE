## 4) 회의 내용(Notes)

### 4.1 수행일지 확정
- **주간 수행일지 작성 포맷 확정**
  - `"금주 수행 내용 / 차주 계획"` 형식
  - 산출물(문서 링크, 보드 링크, ERD 캡처 등) **첨부를 원칙**으로 함
- 작성/공유 위치는 **Confluence 페이지로 통일**

---

### 4.2 요금 상품 데이터 전처리 확정
- 정산 로직에 사용할 **요금제/상품 데이터 전처리 범위 확정**
- 전처리 목적
  - 필수 컬럼 정리(가격/구간/과금 기준 등)
  - 정산에 필요한 **정규화된 형태로 변환**
  - 누락값/이상치 처리 기준 통일
- 전처리 결과물
  - DB 적재 가능 형태(**CSV/테이블**)로 준비
  - ERD 설계에 반영

---

### 4.3 ERD 관련 논의
- 현재 도메인 모델을 기반으로 **ERD 엔티티/관계 구조 정리**
- 특히 설계 포인트:
  - **정산 이력(대용량)**, **발송 이력(상태 추적)** 테이블은
    조회 패턴 및 트리거 기준으로 최적화 방향 고려
    - 인덱스 / 파티셔닝 등
- ERD 운영 방식
  - **설계 확정본**과 **추후 변경 가능 항목**을 구분하여 관리

---

### 4.4 Bounded Context 수정 논의
- EventStorming 결과를 바탕으로 **컨텍스트 경계가 애매한 부분 재검토**
- 원칙 재확인
  - **정산(배치)**과 **발송(Kafka)** 영역 책임 경계 명확화
  - 연계되는 부분은 **Policy(이벤트 기반 후속 처리)**로 연결
- 수정된 Bounded Context 공유
  - 팀 공용 문서로 공유
  - ERD/모델링에 반영

---

### 4.5 설계 세부 일정 확정
남은 설계 일정은 다음 순서로 진행한다.
1) Domain Modeling 확정  
2) ERD 확정  
3) Infra Architecture 설계  
4) 구현 착수(배치/발송 우선순위 결정)

- 각 단계별 완료 기준(DoD)을 간단히 합의하고,
- Jira 스프린트/이슈로 연결한다.
