# Backend (Spring/Java) Code Review Style Guide

## 1) 코드 품질
- 공개 API(Controller)는 DTO로만 입출력하고 Entity 직접 노출 금지
- 메서드/변수 이름은 역할이 드러나게 작성 (약어 남발 금지)
- 매직넘버/문자열 상수는 상수화

## 2) 예외/에러 처리
- RuntimeException 남발 대신 도메인 예외 클래스로 의미를 분리
- 클라이언트 오류(4xx) vs 서버 오류(5xx) 구분해서 응답

## 3) 트랜잭션/DB
- 조회 전용은 readOnly=true 고려
- N+1 가능성 있으면 근거와 함께 지적

## 4) 테스트
- 신규 로직은 최소 단위 테스트 1개 이상
- 경계값/예외 케이스 1개 이상 포함

## 5) 리뷰 코멘트 스타일
- "왜 문제인지" + "어떻게 고치면 좋은지"를 함께 제시
